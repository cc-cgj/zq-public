

	一、概念

		1、key 特殊 attribute 主要用做 Vue 的虚拟 DOM 算法的提示，以在比对新旧节点组时辨识 VNodes。

		2、如果不使用 key，Vue 会使用一种算法来最小化元素的移动并且尽可能尝试就地修改/复用相同类型元素。

		3、而使用 key 时，它会基于 key 的顺序变化重新排列元素，并且那些使用了已经不存在的 key 的元素将会被移除/销毁。

	二、类型

		number | string | symbol

	三、注意

		1、有相同父元素的子元素必须有唯一的 key。重复的 key 会造成渲染错误。

		2、不要使用对象或数组之类的非基本类型值作为 v-for 的 key。请用字符串或数值类型的值。

	三、例
		1、最常见的用例是和 v-for 一起使用：

			<ul> <li v-for="item in items" :key="item.id">...</li> </ul>

		
		2、它也可以用于强制替换元素/组件而不是重复使用它。当你遇到如下场景时它可能会很有用：

			完整地触发组件的生命周期钩子

			触发过渡

				例如：

					<transition> <span :key="text">{{ text }}</span> </transition>
				
					当 text 发生改变时，<span> 总是会被替换而不是被修改，因此会触发过渡。
